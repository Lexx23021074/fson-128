Як зробити правильно (Алгоритм переходу)
Найкраще використовувати термінал VS Code. Це швидше, ніж будь-які маніпуляції в браузері чи файловому менеджері:

Створення копії попереднього уроку:
Відкрийте термінал у VS Code (переконайтеся, що ви в папці html) і введіть:

Bash
cp -r module-02/lesson-1 module-02/lesson-2
Це миттєво створить папку lesson-2 з усім кодом, картинками та стилями з lesson-1.

Фіксація в Git (Commit):
Тепер важливо одразу зафіксувати, що ви почали новий урок, щоб не плутати зміни:

Bash
git add module-02/lesson-2
git commit -m "init: start lesson-2 as continuation of lesson-1"
git push

Ось як це виправити зараз і як робити правильно надалі, щоб отримувати «чисту» копію.

1. Як виправити те, що вже створено
Якщо у вас зараз шлях виглядає як module-02/lesson-2/lesson-1/..., виконайте ці команди в терміналі:

Bash
# Переміщуємо вміст вкладеної папки на рівень вище
mv module-02/lesson-2/lesson-1/* module-02/lesson-2/

# Видаляємо тепер вже порожню вкладену папку lesson-1
rm -rf module-02/lesson-2/lesson-1
2. Правильний алгоритм для майбутніх уроків
Щоб створити нову папку lesson-2, яка буде ідентичною до lesson-1, але без вкладеності, використовуйте таку команду (перебуваючи в корені репозиторію html):

Варіант А (найпростіший):

Bash
cp -r module-02/lesson-1 module-02/lesson-2
Важливо: папка lesson-2 ще не має існувати в цей момент. Тоді команда створить нову папку lesson-2 і покладе в неї вміст lesson-1.

Варіант Б (якщо папка вже створена):

Bash
cp -r module-02/lesson-1/. module-02/lesson-2/
Зверніть увагу на крапку після lesson-1/. Це означає: «скопіюй тільки вміст цієї папки в цільову папку».

3. План для наступних модулів
Коли ви перейдете до module-03, алгоритм буде таким:

Створюєте папку нового модуля:

Bash
mkdir module-03
Копіюєте туди останній напрацьований код (наприклад, з module-02/lesson-2):

Bash
cp -r module-02/lesson-2 module-03/lesson-1
Підсумок (Золоте правило):
Якщо хочете копіювати папку як нову сутність — використовуйте cp -r source destination (де destination ще не існує).

Якщо хочете скопіювати тільки нутрощі в уже готову папку — використовуйте cp -r source/. destination/.

Тепер у вас у module-02/lesson-2 буде чистий код, на якому ви зупинилися в першому уроці, і ви зможете продовжувати проект. Не забудьте після виправлення зробити git add . та git commit, щоб зберегти правильну структуру в GitHub.
